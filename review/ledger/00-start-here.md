## Summary of issues found in this file

1. **"over any transport"** (line 13) is the highest-risk claim — it presents the proposed SBD/LoRa transports as part of a delivered "any transport" capability. Only QUIC/Iroh, BLE, and embedded-UDP ship. This must be qualified at the very first sentence a reader sees.
2. **"the five repos"** undercounts — there are six code repos (umbrella `peat` + mesh/btle/lite/gateway/node), and `peat-protocol` is a crate within the umbrella, distinct from `peat-node`.
3. **"reading the actual source in this folder"** overstates verifiability — the leaf-transport internals were not all present in a single tree and were corroborated via re-export shims/tests/gbrain per the ground-truth audits.
4. **peat-mesh "node binary"** mis-attributes the deployable node binary; that is `peat-node`.
5. **256 KB** for peat-lite and the **"~10 min / 3-node"** quickstart are unenforced/unverified figures that should be flagged, not stated as facts.
6. **"AI models"** as a connected system type is vision, not a shipped integration surface.

No FIPS or vendor-name house-rule violations are introduced in this file (it names Defense Unicorns as the project owner, which is the project itself, not a third-party consumer; no ChaCha20 references). The data-plane/control-plane distinction for the gateway should be reinforced in the sensor→command-post framing.
