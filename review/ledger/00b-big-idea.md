# Claim Ledger — `learning/00b-the-big-idea.md` (The Big Idea)

**Auditor stance:** adversarial. Source of reality = `learning/review/ground-truth/*.md` (peat, peat-mesh, peat-btle, peat-lite, peat-gateway, peat-node) plus the whitepaper at `peat/docs/whitepaper/`.

**Framing context that softens many verdicts:** this module explicitly declares itself a *distillation of the whitepaper* (lines 3-6) and repeatedly tells the reader the whitepaper is a "positioning + vision document" whose performance numbers are "lab validation," not field-proven (lines 183-184, 213-216). That honesty is real and earns partial credit. The remaining problem is that the module frequently presents whitepaper marketing figures and design-intent vocabulary as if they map cleanly onto **shipped code**, when the ground truth shows the code (a) does not implement several of them, (b) uses different names, and (c) sits behind ADRs that are still *Proposed*. There is **no shipped/in-flight/proposed/speculative labeling** anywhere in the module — a §11 Definition-of-Done failure on its own.

## Summary of the most damaging issues (priority order)

1. **Vocabulary heads-up is factually wrong (L147-151)** — drops `Platform` from the 5-tier ADR-066 set, presents the rename as done ("current code") when ADR-066 is **Proposed** and the rename is partially landed (peat-mesh leaf still `Node`; peat-btle still fully legacy Squad/Platoon/Company). This is the single claim a skeptical staff engineer with `peat-btle/src/lib.rs` open will catch immediately. *(Cross-refs: peat GT §4/§10, peat-mesh GT §4, peat-btle GT §4.)*

2. **"Trust as data = replicated CRDT authority state" (L130-131) presented as fact** — it is design intent; the authorization model is deferred (peat#941 open). Speculative-unlabeled.

3. **No shipped/in-flight/proposed/speculative labels anywhere** — violates the curriculum's own Definition of Done (§11). Even where claims are correct (Negentropy, Automerge, deterministic election), the reader can't tell what is built vs. argued.

4. **The 500,000 vs ~4,000 connection figure (L82)** quotes the whitepaper's looser depth-4 analytic model while the ground truth trusts the same paper's 9,120/384 lab table — a ~10× internal disconnect the module hides.

5. **"Deep integration = peat-lite" (L211) overstates peat-lite** — it is a codec + bounded CRDTs with no transport or sync engine (peat-lite GT §1/§11).

6. **Spec path `peat/spec/draft-peat-protocol-00.md` (L193) unverified** — confirm it exists before shipping.

**What the module gets right and should keep:** the CRDTs-vs-consensus section (§6) is solid and traces to shipped code (Automerge + Negentropy + deterministic election); the dual 95-99% / 93-99% citation honestly surfaces the whitepaper's own inconsistency; the TRL-4/5 and "lab not field" caveats are exactly the right posture for a defense-prime audience. The module's core sin is not fabrication — it is *faithfully reproducing a vision document without translating it into shipped-vs-proposed reality*, plus one genuinely wrong vocabulary mapping.
